# Reproducibility Statement

This repository supports reproducibility through code organization, workflow documentation, and environment specification while respecting confidentiality restrictions on the experimental data.

## Reproducibility components

The repository is intended to document:

- the overall analysis workflow
- key scripts used for the Streamlit interface and supporting utilities
- expected input-table structures
- software dependencies
- synthetic example data for structural testing
- documentation of data-availability limitations

## What can be reproduced from this repository

Without access to the confidential experimental dataset, users can:

- inspect the software structure
- understand the expected input formats
- run the app using synthetic or approved processed data
- review the analysis workflow
- reproduce the interface-level behavior on example inputs

## What cannot be reproduced publicly

The main thesis results cannot be fully regenerated from this public repository alone because the raw and full processed experimental data are confidential and not publicly distributed.

## Thesis methods wording

Suggested wording for the thesis:

> To support reproducibility, the Streamlit application and associated documentation were organized in a GitHub repository approved for sharing by the host laboratory. Because the experimental data are confidential and belong to an ongoing research line, raw data and full analysis tables were not made publicly available. The repository therefore documents the computational interface, software requirements, and workflow structure, while access to the underlying experimental data remains restricted.
